# Example WASM Plugin: Session Logger (Go)

## Purpose
Demonstrates building a Chronicle WASM logic extension in Go using the
Extism Go PDK. Shows KV storage, calendar integration, and hook handling.

## Capabilities Used
- `log` — server-side logging
- `entity_read` — look up entity details
- `calendar_write` — create calendar events
- `kv_store` — per-plugin key-value storage

## Hooks
- `entity.created` — logs entity creation to KV store
- `entity.updated` — logs entity updates to KV store

## Exported Functions
- `on_hook` — hook event handler (logs activity to KV store)
- `get_activity_log` — returns stored activity log from KV
- `create_session_summary` — creates calendar event summarizing activity
- `on_message` — plugin-to-plugin message handler

## Build
```bash
# With TinyGo (smaller binary):
tinygo build -o dist/session_logger.wasm -target wasip1 main.go

# With Go 1.24+ (native):
GOOS=wasip1 GOARCH=wasm go build -o dist/session_logger.wasm main.go
```

## Files
- `manifest.json` — extension manifest declaring the WASM plugin
- `go.mod` — Go module config with extism go-pdk dependency
- `main.go` — plugin implementation with all exported functions
