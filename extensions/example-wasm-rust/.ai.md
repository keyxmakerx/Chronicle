# Example WASM Plugin: Auto Tagger (Rust)

## Purpose
Demonstrates building a Chronicle WASM logic extension in Rust using the
Extism PDK. Shows hook handling, host function calls, and exported functions.

## Capabilities Used
- `log` — server-side logging
- `entity_read` — look up entity details
- `tag_read` — list campaign tags
- `tag_write` — set entity tags

## Hooks
- `entity.created` — triggers auto-tagging based on entity type

## Exported Functions
- `on_hook` — hook event handler (auto-tags new entities)
- `roll` — pure dice roller (NdS+M format)
- `on_message` — plugin-to-plugin message handler

## Build
```bash
cargo build --release
cp target/wasm32-unknown-unknown/release/chronicle_auto_tagger.wasm dist/auto_tagger.wasm
```

## Files
- `manifest.json` — extension manifest declaring the WASM plugin
- `Cargo.toml` — Rust project config with extism-pdk dependency
- `.cargo/config.toml` — sets wasm32-unknown-unknown as default target
- `src/lib.rs` — plugin implementation with all exported functions
