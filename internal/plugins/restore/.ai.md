# Restore Plugin

## For humans

Admin-facing UI for restoring a previous backup. Mirror of `backup/`: the actual restore mechanics live in `scripts/restore.sh` (POSIX shell); this package shells out to it under a timeout and single-flight lock.

Stood up as part of 0.0.1 Release Readiness (2026-04-25) alongside `backup/`, the deployment runbook, and the `mariadb-client` Dockerfile addition. Restoring is destructive (drops + re-applies DB state); the UI has confirmation prompts + the script verifies the manifest's sha256 before proceeding.

## For AI sessions

### Key files
- `service.go` — single-flight `Run()` wrapper around `scripts/restore.sh`
- `handler.go` — admin HTTP handlers
- `routes.go` — `RegisterRoutes(admin, h)` mounts everything under `/admin`
- `restore.templ` — admin restore UI

### Routes
- `GET /admin/restore` — restore dashboard
- `POST /admin/restore/run` — trigger restore (requires confirmation token)

### Wiring
Constructed inline in `internal/app/routes.go` and registered via `restore.RegisterRoutes(adminGroup, restoreHandler)`. Admin-only; auth enforced by parent admin group. Sensitive operations may add `auth.RequireReauth` per the security checklist.

### Dependencies
- `apperror` — domain error types
- `scripts/restore.sh` at runtime — see `docs/deployment.md` for the script's contract
