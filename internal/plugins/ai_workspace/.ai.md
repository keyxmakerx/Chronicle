# ai_workspace plugin

## Purpose

Chronicle's AI co-pilot surface. Owns:

- **AI Export** — the markdown-for-AI generator (relocated from
  `internal/aiexport/` in C-AI-WORKSPACE-V1-B; functionally identical
  to PR #350)
- **Prompt builder** — Phase 3 (`C-AI-WORKSPACE-V1-C`)
- **AI Import** — Phases 4-5 (`C-AI-WORKSPACE-V1-D` + `-V1-E`)

First plugin added under the post-NW-2.2 plugin-isolation rules.
Doubles as a showcase of the pattern: own settings tab via
`campaigns.RegisterSettingsTab`, own routes via
`RegisterOwnerRoutes`, owns no shared state beyond the renderer
service.

## Tier

**Plugin** under `internal/plugins/ai_workspace/`. Cross-cutting in
nature — depends on every other plugin's lister interface via the
nested `aiexport/` sub-package, but exports nothing visible to those
plugins. The campaigns plugin is the only consumer (via the
SettingsTab registry and the parent route group).

## Wiring

`internal/app/routes.go` does the construction at startup:

```go
aiWorkspaceRenderer := aiexport.NewService(entityService, noteSvc, ...)
aiWorkspaceHandler := ai_workspace.NewHandler(aiWorkspaceRenderer)
aiWorkspaceHandler.SetAuditLogger(&aiWorkspaceAuditAdapter{svc: auditService})

// Contribute the settings tab (slot 55 — between AI Export's old
// slot 50 and Activity's slot 60; pinned in V1-A's
// TestRegisterSettingsTab_MergesAndSorts).
campaignHandler.RegisterSettingsTab(aiWorkspaceHandler.SettingsTabFactory())

// Mount routes on a /campaigns/:id group with auth + membership.
aiWorkspaceCampaignAuthed := e.Group("/campaigns/:id",
    auth.RequireAuth(authService),
    campaigns.RequireCampaignAccess(campaignService),
)
ai_workspace.RegisterOwnerRoutes(aiWorkspaceCampaignAuthed, aiWorkspaceHandler,
    campaigns.RequireRole(campaigns.RoleOwner))
```

## Auth chain

Parent group: `auth.RequireAuth` + `campaigns.RequireCampaignAccess` (membership). Per-route: `RequireRole(RoleOwner)`. AST pin in `internal/wire/ai_export_route_test.go` makes the role gate load-bearing — a future refactor that drops it fails pinpointed.

## SEC-6-AMENDED inheritance

The relocated `aiexport/` sub-package preserves PR #349's invariant: every HTML field funnels through `aiexport.htmlToMarkdown`, which applies `sanitize.HTMLPtr` BEFORE the markdown converter. The AST structural pin `TestRenderers_FunnelThroughHtmlToMarkdown` moved with the package and continues to enforce.

## D4=(c) carve-out

Zero edits to `internal/app/export_adapters.go`, `internal/plugins/campaigns/export_handler.go`, or `internal/plugins/restore/`. AI Export is the lossy AI-facing egress; the lossless backup pipeline remains the round-trip-safe surface.

## Settings tab integration

`SettingsTabFactory` returns a `func(*campaigns.CampaignContext) campaigns.SettingsTab` factory. Campaigns invokes it per-request inside `visibleSettingsTabs`, so the tab's `Content` closure can bind `cc.Campaign.ID` into the form's `hx-get` URL. Slot 55 (between AI Export's retired slot 50 and Activity's slot 60); the V1-A test `TestRegisterSettingsTab_MergesAndSorts` already pinned this slot expecting V1-B to fill it.

## Audit

Each successful `GenerateAIExport` invocation fires `campaign.ai_export.generated` with privacy mode + category count + GM-notes opt-in flag + markdown byte count. No markdown body in the audit (audit log is events, not bulk content). Audit hook is optional via `SetAuditLogger`; nil disables emission for test fixtures.
