# aiexport package

## Purpose

Renders a campaign owner's content into a single markdown document
suitable for pasting into AI tools (Claude, ChatGPT, NotebookLM).
Intentionally lossy — markdown is the wire format for human/AI
reading, not a backup substitute. The lossless backup pipeline at
`internal/app/export_adapters.go` is a separate surface (D4=(c)
carve-out) and is NOT touched by this package.

Implements C-AI-EXPORT-V1 per:
- `cordinator/decisions/2026-05-26-ai-export-pipeline-design.md` (locked design)
- `cordinator/reports/chronicle/2026-05-26-c-ai-export-scoping.md` (§4 file plan, §5 acceptance)

## Tier

**Cross-cutting** (same tier as `internal/middleware/`). Imports plugin
Service interfaces; defines no plugin-specific concerns of its own.

## SEC-6-AMENDED invariant (load-bearing)

**Every user-typed HTML field that the renderer emits MUST funnel
through `htmlToMarkdown`.** That function is the single point that:

1. Calls `sanitize.HTMLPtr(p)` to strip `<script>` / `javascript:` /
   `on*` handlers via bluemonday's UGC policy (same allowlist as the
   `syncapi/egress_sanitize.go` helpers).
2. Hands the sanitized HTML to the `html-to-markdown` converter.

`renderer_test.go::TestRenderers_FunnelThroughHtmlToMarkdown` AST-walks
`renderer.go`, asserts every named helper (`renderEntity`,
`renderNoteTree`, `renderCalendarEvent`, `renderSession`, `renderTimeline`)
contains a `htmlToMarkdown(` call AND does NOT bypass via direct
`getConverter()` access. A future refactor that drops the call fails
the test pinpointed.

If you add a new renderer, also add it to the test's `required` slice.

## Privacy model

Three modes (operator-toggled in PR-B's settings UI):

| Mode | Plugin lister role | Renderer additional filter |
|---|---|---|
| `PrivacyModeSafe` (default) | `RolePlayer` (dm_only stripped at listing) | drop dm_only tags + dm_only calendar events + dm_only timelines/events; skip entity PlayerNotesHTML; skip session GM notes regardless of IncludeSessionGMNotes |
| `PrivacyModePermitted` | `RoleOwner` (owner-view) | renderer trusts listing output; session GM notes respected via IncludeSessionGMNotes flag |
| `PrivacyModeEverything` | `RoleOwner` | renderer trusts listing output; no additional filter |

The renderer never reimplements the visibility model — it delegates
to existing plugin services via the role argument.

## Resilience — the export never blank-errors (2026-07 fix)

The generator degrades instead of failing. Two layers:

1. **Field level (`bodyOrSkip`)** — `htmlToMarkdown` can genuinely error
   (e.g. `golang.org/x/net/html`'s "open stack of elements exceeds 512
   nodes" on deeply-nested / malformed markup). Every renderer used to
   `return` that error, which aborted the **entire** export. Now each site
   runs `htmlToMarkdown(...)` then `bodyOrSkip(...)`, which logs and swaps in
   `convertSkipMarker` for that one field. This honors the long-standing
   docstring intent ("one entity's bad HTML shouldn't poison the whole
   document"). The `htmlToMarkdown(` call stays inline so the SEC-6-AMENDED
   AST pin is unaffected.
2. **Category level (`Generate`)** — a lister/DB error from one category is
   logged + replaced with `categorySkipNote` + skipped; the rest still
   render. (The calendar branch already swallowed errors; this makes every
   category consistent.)

**Why the bug looked owner-/"Everything"-specific:** private / `dm_only`
entities and `PlayerNotesHTML` render **only** in Permitted/Everything mode.
A single such field carrying converter-hostile HTML aborted "export
everything" while Safe mode (which filters that content out) succeeded.
Both modes share one backend path (`RoleOwner`, no extra filter), so this
was never a privacy-value branch — it was the extra content those modes
include. See `resilience_test.go`.

**Entity paging:** `entityService.List` clamps `PerPage` to 100 (and
silently defaults anything larger to 24), so the old single PerPage:10000
request truncated the export to 24 entities. `listAllEntities` pages at
PerPage:100 (dedup by ID, bounded by `exportMaxPages`) so "everything" is
actually everything.

## Locked operator decisions (2026-05-26 AskUserQuestion)

1. **Maps** — DEFER to v2. No `RenderMaps`. `internal/plugins/maps/`
   is not imported.
2. **Bidirectional relations** — render on BOTH endpoints (mild
   duplication accepted; AI-consumer clarity wins). Caller (Service
   orchestrator) queries `RelationLister.ListByEntity` per entity.
3. **Token-count header** — `RenderHeader` emits a `**Estimated
   tokens:** ~__TOKEN_COUNT__` placeholder; `substituteTokenCount`
   replaces it after the full body renders. 4 chars/token heuristic.
