# Backup Plugin

## For humans

Admin-facing UI for triggering whole-database backups and inspecting prior runs. The actual backup mechanics live in `scripts/backup.sh` (POSIX shell, manifest-driven with sha256 + migration version + chronicle version pairs). This package shells out to that script under a timeout and single-flight lock so concurrent admin clicks can't trigger overlapping backup runs.

Stood up as part of 0.0.1 Release Readiness (2026-04-25) alongside `restore/`, the deployment runbook, and the `mariadb-client` Dockerfile addition.

## For AI sessions

### Key files
- `service.go` — single-flight `Run()` wrapper around `scripts/backup.sh`
- `handler.go` — admin HTTP handlers for the backup dashboard
- `routes.go` — `RegisterRoutes(adminGroup, h)` mounts everything under the existing `/admin` group
- `backup.templ` — admin dashboard UI

### Routes
- `GET /admin/backup` — backup dashboard
- `POST /admin/backup/run` — trigger backup
- `GET /admin/backup/list` — JSON list of prior runs

### Wiring
Constructed inline in `internal/app/routes.go` and registered via `backup.RegisterRoutes(adminGroup, backupHandler)`. Admin-only; auth is enforced by the parent admin group.

### Dependencies
- `apperror` — domain error types
- The shell script (`scripts/backup.sh`) at runtime — see `docs/deployment.md` for the script's contract
