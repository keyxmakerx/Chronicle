# Designlab Plugin

## For humans

Admin-only "Design Lab" page. It hosts the **dynamic-surface demo**: a live
character sheet assembled at runtime by the frame (`Chronicle.surface`,
`static/js/widgets/dynamic_surface.js`) from a declarative schema, with the box
bodies supplied by a sample "System" (`static/js/widgets/surface_demo.js`). It
shows off the frame's primitives — expand/collapse boxes, an action that pushes
an overlay, and the mini→full launch (container-transform) — against seeded
(offline) data, so you can SEE the engine before any real System adopts it.

It formerly hosted a static component catalogue ("Demo Page"); the moving
dynamic-surface engine is the demo now (the static mocks were never wired to a
real feature).

Not user-facing; not addon-gated; only reachable via `/admin/design-lab`.

## For AI sessions

### Key files
- `handler.go` — single page handler rendering the demo
- `routes.go` — `RegisterRoutes(adminGroup, h)` mounts it under `/admin`
- `design_lab.templ` — `SurfaceDemoPage()`: the page shell + mount targets
  (`#surface-demo-root`, `#surface-demo-mini`) and the `surface_demo.js` include
- The actual sheet content lives in the two widget JS files above, NOT here —
  the frame builds the boxes, the sample System renders the bodies.

### Routes
- `GET /admin/design-lab` — the dynamic-surface demo page

### Wiring
Constructed inline in `internal/app/routes.go`; registered via `designlab.RegisterRoutes(adminGroup, designLabHandler)`. Admin-only.

### Dependencies
- None beyond shared templ layouts and the global CSS layers (see `.ai/conventions.md` §"CI tenet-enforcement guards" + the @layer architecture from PR #329)
