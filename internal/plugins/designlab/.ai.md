# Designlab Plugin

## For humans

Admin-only UI component showcase ("Design Lab") for previewing button variants, cards, badges, alerts, form inputs, typography, and other design-system components without affecting the live site. Used during UI work to verify new patterns against the shared component library before applying them to a real page.

Not user-facing; not addon-gated; only reachable via `/admin/designlab`.

## For AI sessions

### Key files
- `handler.go` — single page handler rendering the showcase
- `routes.go` — `RegisterRoutes(adminGroup, h)` mounts everything under `/admin`
- `designlab.templ` — the showcase page (all components inline)

### Routes
- `GET /admin/designlab` — the showcase page

### Wiring
Constructed inline in `internal/app/routes.go`; registered via `designlab.RegisterRoutes(adminGroup, designLabHandler)`. Admin-only.

### Dependencies
- None beyond shared templ layouts and the global CSS layers (see `.ai/conventions.md` §"CI tenet-enforcement guards" + the @layer architecture from PR #329)
