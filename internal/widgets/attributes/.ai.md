# Attributes Widget

## Purpose

Displays and edits entity custom fields (attributes) defined by entity type
templates. Supports per-entity field overrides: users can add extra fields,
hide inherited fields, or modify field definitions without altering the
category-level template.

## Tier

**Widget** -- reusable UI block with API-driven frontend.

## Files

| File | Role |
|------|------|
| `internal/widgets/attributes/templates/` | Templ mount-point (renders `data-widget="attributes"`) |
| `static/js/widgets/attributes.js` | Client-side widget implementation |

## Dependencies

- **Uses:** Entity fields API (`GET/PUT /campaigns/:id/entities/:eid/fields`)
- **Uses:** Entity field overrides API (`PUT/DELETE .../field-overrides`)
- **Used by:** Entity profile page (entities plugin)

## Frontend Widget

### Mount Syntax

```html
<div data-widget="attributes"
     data-endpoint="/campaigns/{cid}/entities/{eid}/fields"
     data-editable="true"
     data-csrf-token="{token}">
</div>
```

### Configuration

| Attribute | Required | Description |
|-----------|----------|-------------|
| `data-endpoint` | Yes | Fields API endpoint (GET/PUT) |
| `data-editable` | No | Enable editing (`true`/`false`) |
| `data-csrf-token` | No | CSRF token for mutations |

### Modes

1. **Read-only display** -- shows field labels and formatted values (URLs as links, booleans as Yes/No).
2. **Edit form** -- groups fields by section with type-appropriate inputs (text, number, url, textarea, select, checkbox).
3. **Customize panel** -- per-entity field overrides: toggle visibility, add custom fields, reset to template.

### API Calls

- `GET {endpoint}` -- loads `{ fields, type_fields, fields_data, field_overrides }`
- `PUT {endpoint}` -- saves `{ fields_data: { key: value } }`
- `PUT {endpoint → /field-overrides}` -- saves per-entity overrides
- `DELETE {overrides-endpoint}` -- resets to category template
